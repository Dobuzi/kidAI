# kidAI Web Learning Design

## Goal

어린이를 위한 웹 기반 AI 학습 도우미의 첫 버전을 만든다.

핵심 목표:
- 초등 범위의 국어, 영어, 수학, 과학 학습 지원
- 자유 질문과 문제 풀이를 한 화면 흐름 안에서 함께 제공
- 연령 전체를 대상으로 하되 설명 난이도를 조절

비목표:
- 로그인/계정 시스템
- 서버 저장소
- 학부모 대시보드
- 결제, 멀티유저, 랭킹

## Product Shape

형태:
- 웹 앱

핵심 구조:
- 혼합형 대시보드
- 홈에서 과목 선택, 추천 문제, 자유 질문 진입을 함께 제공

대상 사용자:
- 전 연령 어린이

연령 대응 방식:
- 별도 연령 가입 없이 난이도 선택으로 설명 수준을 조절
- 난이도 옵션은 `쉬움`, `보통`, `도전`

## User Experience

### Home

홈 화면은 다음 요소만 포함한다.

- 과목 카드 4개: 국어, 영어, 수학, 과학
- `오늘의 질문` 입력 영역
- `추천 문제` 카드 1개
- 최근 학습 기록 영역

의도:
- 첫 화면에서 바로 질문하거나 문제를 시작할 수 있어야 한다.
- 기능을 넓히지 않고 학습 시작 지점을 명확히 둔다.

### Learning Screen

학습 화면은 2영역 구조로 둔다.

- 왼쪽: 문제, 주제, 과목 정보
- 오른쪽: AI 설명과 대화

사용 흐름:
1. 사용자가 과목 또는 추천 문제를 선택한다.
2. 난이도를 선택한다.
3. AI가 바로 정답만 주지 않고 힌트부터 제공한다.
4. 필요하면 추가 질문을 이어간다.

## AI Behavior

응답 원칙:
- 어린이 친화적 톤
- 답만 즉시 제공하지 않음
- `힌트 -> 풀이 설명 -> 정리` 순서 유지
- 과도하게 어려운 표현 지양

난이도별 차이:
- 쉬움: 짧은 문장, 쉬운 단어, 단계 수 적음
- 보통: 표준 설명, 예시 포함
- 도전: 스스로 생각할 질문 추가

안전 장치:
- 유해하거나 부적절한 요청 차단
- 학습 범위를 벗어난 민감 요청은 거절 또는 우회
- 아이에게 혼란을 줄 수 있는 공격적 표현 금지

## Scope for V1

포함:
- 홈 화면
- 과목 4종
- 난이도 선택
- 추천 문제 1개 표시
- 자유 질문 입력
- 대화형 AI 응답 UI
- 최근 학습 기록의 로컬 저장

제외:
- 회원가입/로그인
- 서버 DB
- 이미지 생성
- 음성 입출력
- 숙제 제출/채점 시스템
- 관리자 기능

## Technical Direction

권장 스택:
- Next.js
- React
- TypeScript

상태 관리:
- 초기 버전은 로컬 상태 + `localStorage`

데이터 전략:
- 실제 AI 연결 전에도 동작하도록 과목별 샘플 문제와 샘플 응답 포함
- 이후 OpenAI API 연결이 가능하도록 AI 호출 레이어 분리

초기 구현 단위:
- 정적 샘플 데이터 기반 UI
- AI 응답은 우선 모의 응답 또는 간단한 서버 라우트로 교체 가능 구조

## Success Criteria

다음이 되면 첫 버전 설계 성공으로 본다.

- 홈에서 과목 선택과 질문 시작이 가능하다.
- 사용자가 추천 문제를 시작할 수 있다.
- 학습 화면에서 힌트, 설명, 정리 흐름을 볼 수 있다.
- 난이도에 따라 답변 톤과 깊이가 달라진다.
- 최근 학습 기록이 브라우저에 남는다.

4. 혼합형 대시보드 UI 구현

6. 난이도 반영 응답 로직 연결
7. `localStorage` 기반 최근 학습 기록 저장
8. 이후 실제 AI API 연결 지점 추가
